3 · Source files at a glance

App.mc
App entry point. Extends Application.AppBase. Decides which view to show on launch (resume prompt vs. setup).
SetupView.mc
Pre-match configuration: sets, tiebreak format, server, names. Hands off to MainView when started.
MainView.mc
The live match screen + its InputDelegate. Renders score; routes taps/swipes to the engine.
PostMatchView.mc
End-of-match summary list: duration, sets, winners, errors, double faults, HR.
MatchMenu.mc
In-match menu (end match, undo, settings) built from menu.xml.
TennisMatchEngine.mc
Pure scoring logic. Points → games → sets, tiebreak / super TB, undo stack. No UI, no I/O.
TennisActivityManager.mc
Wraps ActivityRecording + FitContributor. Starts/stops the session, writes custom FIT fields per point.
MatchPersistence.mc
Save / load match snapshots through Toybox.Storage. Powers the "Resume match?" prompt.
resources/*.xml
Strings, menu items, drawables, layout, manifest (appID, permissions, device targets).

6 · Quick glossary

Monkey C
Garmin's app language — strongly typed, garbage collected, runs on a small VM on the watch.
Connect IQ
Garmin's app SDK and platform. Version 9.1.0 here.
API Level 6.0
Feature/version contract the app declares against. Determines which Toybox APIs are available.
Toybox
Namespace for Garmin's standard modules (WatchUi, Graphics, Storage, ActivityRecording, etc.).
View
A drawable screen. Implements onUpdate(dc) using the Graphics API.
InputDelegate
The companion object to a View. Receives taps, swipes, and key presses.
FitContributor
API for writing custom fields into the activity's FIT file (e.g. score, winners).
ActivityRecording
Starts an "Other" activity session so HR, time, and FIT fields are saved.
.prg / .iq
Compiled bytecode for the device; the .iq is the bundle uploaded to the store.
